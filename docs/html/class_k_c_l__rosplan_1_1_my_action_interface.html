bool KCL_rosplan::MyActionInterface::concreteCallback ( const rosplan_dispatch_msgs::ActionDispatch::ConstPtr &  msg)

A callback function of MyActionInterface class'.

Parameters
msgtasks arguement of type 'rosplan_dispatch_msgs::ActionDispatch'
Returns
returns a bool value based on the outcome of the assigned task to the action server.

This function works has a callback for the 'goto_waypoint' action in PDDL domain file. During the dispatching of the plan, this function get triggers with the occurance of 'goto_waypoint' action. It contains the low-level controller implementation of 'goto_waypoint' action which made the robot in the Gazebo simulation move from one waypoint to another using a action service. Once the robot has reach its desired waypoint location, it calls two services 'request_set_orientation' and 'request_move_arm' to collect the hints.

void KCL_rosplan::MyActionInterface::updateKB ( const std::string &  req)

This function update knowlegde in the ROSPlan Knowlegde Base (KB)

Parameters
reqTake string as arguement which signify which type of update in KB needs to be made.
Returns
none (function type is void)

This function update the knowlegde in the ROSPlan KB in two situations. Firstly, when the robot has collected hints but they are inconsistent, incomplete and incorrect. In this case, knowlegde about the visited waypoints will be remove from the KB. This way after replanning robot will visit all the waypoints again and collect new hints. Secondly, when the robot has collect hints that are consistent, complete and correct. In this case all the waypoints will be marked as visited so that the robot directly go to wp0 and print the hint statment.
